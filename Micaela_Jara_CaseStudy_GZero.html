Redesign QA Validation for GZero Media

01 — Context

A geopolitical media brand undergoing a full redesign

GZero Media is a leading political risk and geopolitical analysis publication. Its redesign represented a significant investment in brand positioning — a new visual identity, new typography system, updated component library, and improved responsive behavior across devices.

For a redesign at this scale, the gap between what was designed in Figma and what gets implemented in code is where quality breaks down. My role was to close that gap systematically before launch.

💡 A redesign is only as good as its implementation. Pixel-perfect QA is what separates a launch that builds brand trust from one that undermines it.

02 — Problem Discovery

Four categories of implementation failures

Through systematic comparison of the live implementation against Figma specs, I identified failures across four distinct areas:

🔤
Typography
Font mismatches

Wrong font families, weights, or sizes applied to headings, body text, and UI labels — deviating from the defined type system.

📐
Sizing
Incorrect spacing and dimensions

Elements rendered at wrong sizes, with incorrect padding, margins, or proportions compared to the Figma design specs.

📱
Responsive
Layout failures on mobile and tablet

Components breaking at smaller breakpoints — elements overflowing, stacking incorrectly, or not adapting as specified in the responsive Figma frames.

🖱️
Interaction
Broken buttons and interactive states

CTAs and buttons not functioning correctly — wrong hover states, broken links, or interaction failures that blocked user actions.

03 — Process

Figma-to-live systematic validation

1
Reviewed Figma specs in detail before testing

I studied the redesign specs — typography scales, component states, spacing rules, and responsive frames — to build a clear mental model of expected behavior before touching the live environment.

2
Side-by-side Figma vs. implementation comparison

I methodically compared each component and page section against its Figma reference, using DevTools to inspect computed styles and confirm exact values for fonts, sizes, colors, and spacing.

3
Cross-device responsive validation

I tested every page template across mobile and tablet breakpoints, comparing against the responsive Figma frames and documenting deviations in layout, component behavior, and visual hierarchy.

4
Interaction and CTA testing

I validated all interactive elements — buttons, links, navigation items — checking for correct hover states, click behavior, and link destinations across the redesigned components.

5
Documented and prioritized all findings

Every issue was logged in JIRA with Figma references, screenshots, and severity ratings — giving developers a precise diff between spec and implementation to work from.

05 — Product Thinking

What redesign QA taught me about product quality

A redesign is a product bet. The organization has invested in a new direction, and the launch is the moment that bet pays off — or doesn't. QA at this stage isn't just about finding bugs. It's about protecting the integrity of a strategic decision.

Working through GZero's redesign taught me to think in systems: a font mismatch isn't one bug, it's a signal that the typography system wasn't properly translated from design to code. A broken button on mobile isn't isolated — it's a gap in how interactive states were specified or communicated.

💡 The best redesign QA thinks like a PM: what was the intent of this design decision, and does the implementation honor it?

Product Case Study

Fixing a Critical Reservation Failure on Penske's Rental Platform

How a broken API endpoint was discovered through network analysis — and why it mattered for the business.

Company RebelMouse
Client Penske Truck Leasing
Platform Commercial & Personal Rental Sites
Role QA Engineer / Product Collaborator
Focus Area UX Flows · API Integration · Bug Discovery
01 — Context

The product and what was at stake

Penske operates two main customer-facing rental platforms: one for personal truck rentals and one for commercial fleet rentals. These sites are the primary conversion channel — customers go there to browse availability, configure their rental, and complete a booking.

A broken reservation flow doesn't just create a poor user experience. It directly costs the business customers who are ready to pay.

💡 Reservation completion is the most critical user journey on any rental platform. Every drop-off in that funnel is measurable lost revenue.

02 — Problem Discovery

Users couldn't complete the reservation form

During exploratory testing of the reservation flow, I identified that users were unable to submit the final step of the booking form. The process would stall silently — no confirmation, no error message, just a broken state that forced users to abandon.

Rather than assuming it was a frontend issue, I opened the browser's Network tab to inspect what was happening at the infrastructure level.

// Network tab inspection — reservation submit
 
Request: POST /api/reservations/create
Status: 500 Internal Server Error
 
Response body:
{
  "error": "Integration failure — endpoint misconfiguration",
  "data": null
}
 
// Root cause: API endpoint incorrectly integrated,
// returning error response instead of processing reservation

The endpoint responsible for processing reservations was returning a server error on every submission attempt. The frontend had no fallback — it simply froze, giving users no feedback and no path forward.

03 — Process

From discovery to resolution

1
Exploratory testing of the reservation flow

I tested the end-to-end booking journey on both the commercial and personal rental sites, simulating real user behavior across different configurations and form inputs.

2
Network tab analysis to identify root cause

Using browser DevTools, I inspected the network requests fired during form submission. The API call to the reservation endpoint was returning a 500 error, confirming the issue was on the backend integration layer — not the UI.

3
Validated with Postman to isolate the endpoint

I reproduced the error directly in Postman, confirming the endpoint was misconfigured independently of the frontend. This gave the engineering team a clear, reproducible test case to work from.

4
Documented and reported to dev team

I filed a detailed bug report in JIRA, including the endpoint path, response payload, reproduction steps, and severity assessment. The clear documentation allowed the team to prioritize and resolve the issue efficiently.

5
Fix verified and regression tested

After the engineering team corrected the API integration, I ran a full regression pass on the reservation flow across both platforms to confirm the fix and ensure no regressions were introduced.

05 — Product Thinking

What this experience taught me about product

From a Product Manager perspective, this bug was not just a technical issue — it was a conversion problem. The reservation flow is the highest-stakes user journey in any rental product. A silent failure at submission means users don't know what went wrong, can't retry, and most likely leave to a competitor.

This experience reinforced something I carry into every product context: the most critical failures are often invisible to users. Good product quality work means going below the surface — into the network layer, the API responses, the backend logic — to find what's actually breaking the experience.

🧠 A PM who understands how APIs and backend integrations fail is a PM who can have more honest, grounded conversations with engineering about severity, scope, and tradeoffs.
